# CSV-контракт таблицы шага расчёта

Эталонный формат для backend-импорта/экспорта таблиц инженерного практикума.

## Правила

1. Первая строка — наименования граф таблицы через запятую.
2. Каждая следующая строка — данные шага расчёта.
3. Если данных нет, ячейка содержит **один пробел**: `" "`.
4. Число колонок в строке не должно превышать число заголовков.
5. Короткие строки при импорте дополняются до ширины таблицы пробелами.

## Файл примера

- `calculation_steps.csv` — пример с пропусками данных, записанными как пробел.
